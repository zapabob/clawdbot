# 2026-01-30 VRChat Pro v1.2.2 Director's Cut

## 概要

ユーザーからの詳細なフィードバックに基づき、カメラ制御の物理パラメータをVRChat公式仕様に完全準拠させ、ファイル探索の堅牢性を極限まで高めました。これにより「撮影ディレクター」としての実用性が飛躍的に向上しました。

## 修正内容

### 1. 公式準拠のカメラレンジ (Official Camera Ranges)

以前の `0.0 - 1.0` の正規化をやめ、VRChatの物理カメラモデルに合わせた実数制御へ移行しました。

| パラメータ    | 以前の範囲 | **v1.2.2 (公式仕様)** | 単位/備考        |
| :------------ | :--------- | :-------------------- | :--------------- |
| **Zoom**      | 0.0 - 1.0  | **20 - 150**          | FOV (mm換算相当) |
| **Aperture**  | 0.0 - 1.0  | **1.4 - 32**          | F値 (被写界深度) |
| **PhotoRate** | -          | **0.1 - 2.0**         | 撮影間隔         |
| **Duration**  | -          | **0.1 - 60**          | タイマー秒数     |

これにより、`setZoom(50)` のような直感的な指示で、標準レンズ相当の画角を一発で設定可能になりました。もちろん安全装置として、この範囲外の値は自動的にクランプされます。

### 2. 堅牢なファイル探索 (Robust Discovery)

アバター設定ファイル (`.json`) のある `AppData/.../OSC/` ディレクトリ構造は、ユーザーID (`usr_xxx`) によってフォルダ名が変わるため、単純なパス指定では失敗することがありました。
v1.2.2 では `fast-glob` を導入し、`**/Avatars/<avatarId>.json` というパターンマッチングを行うことで、どのユーザープロファイルを使っていても確実に設定ファイルを発見・ロードできるようになりました。

### 3. 手動リロード (Manual Reload)

開発中やテスト時に「今すぐパラメータを再読み込みしたい」というニーズに応え、`/vrc_reload` コマンドを追加しました。エージェントを再起動することなく、現時点のディスク上の最新定義を取り込めます。
