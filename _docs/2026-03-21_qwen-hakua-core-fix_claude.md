# qwen-hakua-core モデル修正ログ

- **日時**: 2026-03-21
- **機能名**: qwen-hakua-core-fix
- **実装AI名**: Claude

## 発生した問題
- **事象1**: ユーザーからの報告「thinkタグだけで会話が成立しない」。
- **事象2**: OpenClawで `reasoning: true` に設定変更後、「無反応になった」。
- **事象3**: 再度設定を戻して確認したところ、「長考しすぎて反応がない」ことが判明。

## 根本原因
指定されたモデル（Qwen3.5-9B-Q8.0版）は9.5GBと非常に重く、さらに `SOUL.md` という強力かつ長文のプロンプトがOllamaのシステムプロンプトに組み込まれていました。
その結果、
1. OOM（VRAM枯渇）を回避するためのメモリ・スワップが発生し、推論速度が極端に低下。
2. モデルが `SOUL.md` の複雑な指示を処理するため、冗長な「思考」や発言を行って長考状態に陥り、タイムアウトが発生。

※ユーザーより「SOUL.md の変更は厳禁」との絶対指令があったため、SOUL.mdの中枢部分は一切改変していません。

## 対応内容
基盤レイヤーのみ（OllamaのModelfile）で問題に対処しました。

1. **`models.json` の設定復元**: `reasoning: false` に戻し、無反応バグを解消。
2. **`scripts/Modelfile.hakua` へのシステム上書き指示追記**:
   - `[System Override: Formatting & Brevity]` セクションを追加。
   - `<think>` タグの出力禁止を明記。
   - **「2〜3行以内（最大100文字程度）で端的に返答すること」** を明記し、長考・冗長な出力を強制的に防止。
3. **コンテキスト制限 (`PARAMETER num_ctx 8192`) の付与**:
   - 推論時のVRAM消費を大幅に削減し、生成速度を圧倒的に高速化。
4. **モデル再構築**:
   - `ollama create qwen-hakua-core -f scripts/Modelfile.hakua` を実行し、モデルをローカルに上書き反映。

以上により、SOUL.mdの規則性を保ったまま、レスポンス速度と対話機能を正常化しました。
