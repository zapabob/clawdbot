## 背景・原因

起動時に以下のエラーが発生していた：

```
Config invalid
File: .openclaw-desktop/openclaw.json
Problem:
  - plugins: plugin: plugin manifest requires configSchema
  - plugins: plugin: plugin manifest requires configSchema

Run: openclaw --profile desktop-stack doctor --fix
```

### バリデーションの仕組み

1. `src/plugins/manifest.ts:178-181` — `loadPluginManifest()` は `configSchema` フィールドが存在しない場合に `{ ok: false, error: "plugin manifest requires configSchema" }` を返す。
2. `src/plugins/manifest-registry.ts:326-332` — マニフェストロードに失敗した場合、`pluginId` なしのエラー診断をキューに積む。
3. `src/config/validation.ts:368-373` — `pluginId` がない診断は path=`plugins`、message=`plugin: {error}` としてフォーマットされる。

これにより `plugins: plugin: plugin manifest requires configSchema` という2件のエラーが表示されていた。

### 原因ファイル

`grep -rL "configSchema" extensions/*/openclaw.plugin.json` で特定：

| ファイル                                           | 問題                          |
| -------------------------------------------------- | ----------------------------- |
| `extensions/universal-skills/openclaw.plugin.json` | `configSchema` フィールドなし |
| `extensions/x-poster/openclaw.plugin.json`         | `configSchema` フィールドなし |

どちらも以前のセッションで新規作成したプラグインで、`configSchema` を付け忘れていた。

## D&D 修正 + sandbox: false

### 背景

Electron 28 で `sandbox: true` がデフォルト化され、renderer 内の `File.path` プロパティが
`<input type="file">` / D&D どちらでも `undefined` になっていた。

```javascript
modelPath = file.path ?? null; // file.path = undefined → null → 早期 return
```

また `app.ts` の D&D ハンドラーは `app.js` が持つ `fileBuffer` 事前読み込みロジックを欠いていた。

### 修正内容

| ファイル                       | 変更                                                        |
| ------------------------------ | ----------------------------------------------------------- |
| `electron/main.ts` / `main.js` | webPreferences に `sandbox: false` 追加                     |
| `renderer/app.ts`              | drop ハンドラーに `fileBuffer` + `loadModel()` ヘルパー追加 |

### sandbox: false について

ローカル専用デスクトップアプリで外部 URL を読み込まないため問題なし。
`file.path` が renderer で再び利用可能になり、D&D / ダイアログ両方が正常動作する。

---

## 今後の注意

新しいプラグイン (`openclaw.plugin.json`) を作成するときは必ず `configSchema` を含めること。設定が不要なプラグインでも空オブジェクトスキーマが必須。

## Live2D コンパニオン: ファイル選択ボタン本番実装

### 背景・問題

- `★ Live2D` / `♦ VRM` / `◈ FBX` の3ボタンは `<input type="file">` を click() するだけのモック実装だった。
- Electron の `contextIsolation: true` 環境では `file.path` が取得できない場合があり、`reloadModel(filePath)` に渡すフルパスが `file.name`（ファイル名のみ）にフォールバックして読み込みが失敗するケースがあった。

### 実装内容

#### 変更ファイル

| ファイル                             | 変更内容                                                             |
| ------------------------------------ | -------------------------------------------------------------------- |
| `electron/main.ts` / `main.js`       | `dialog` をインポート、`open-file-dialog` IPC ハンドラー追加         |
| `electron/preload.ts` / `preload.js` | `openFileDialog()` を `companionBridge` 経由で公開                   |
| `renderer/app.ts` / `app.js`         | `openWithDialog()` + `handleFileDirect()` 追加、ボタンハンドラー置換 |

#### フロー（本番実装後）

```
ボタンクリック
  → openWithDialog(type)
  → companionBridge.openFileDialog({ filters, title })
  → [main.ts] dialog.showOpenDialog() → Windows エクスプローラーが開く
  → ユーザーがファイル選択
  → fs.readFile(filePath) で Buffer 取得
  → { ok, filePath, buffer } をレンダラーに返す
  → handleFileDirect(filePath, buffer, type)
  → VRM/FBX: reloadModelFromBuffer(buffer, filePath)
  → Live2D: reloadModel(filePath)（file:// で読み込み）
```

#### フォールバック

`companionBridge.openFileDialog` が未定義の場合（非 Electron 環境）は従来の `<input type="file">.click()` にフォールバック。ドラッグ&ドロップも引き続き動作。

#### IPC チャンネル

`"open-file-dialog"` — renderer → main invoke。`dialog.showOpenDialog()` を使用。

### filters 定義

| ボタン   | フィルター                  |
| -------- | --------------------------- |
| ★ Live2D | `model3.json`, `model.json` |
| ♦ VRM    | `vrm`                       |
| ◈ FBX    | `fbx`                       |
