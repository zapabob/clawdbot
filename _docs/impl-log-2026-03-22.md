# 実装ログ - 2026-03-22

## 概要

- プラグインマニフェストの `configSchema` 欠落を修正した。
- `extensions/universal-skills` および `extensions/x-poster` の `openclaw.plugin.json` に空の `configSchema` を追加。

---

## 背景・原因

起動時に以下のエラーが発生していた：

```
Config invalid
File: .openclaw-desktop/openclaw.json
Problem:
  - plugins: plugin: plugin manifest requires configSchema
  - plugins: plugin: plugin manifest requires configSchema

Run: openclaw --profile desktop-stack doctor --fix
```

### バリデーションの仕組み

1. `src/plugins/manifest.ts:178-181` — `loadPluginManifest()` は `configSchema` フィールドが存在しない場合に `{ ok: false, error: "plugin manifest requires configSchema" }` を返す。
2. `src/plugins/manifest-registry.ts:326-332` — マニフェストロードに失敗した場合、`pluginId` なしのエラー診断をキューに積む。
3. `src/config/validation.ts:368-373` — `pluginId` がない診断は path=`plugins`、message=`plugin: {error}` としてフォーマットされる。

これにより `plugins: plugin: plugin manifest requires configSchema` という2件のエラーが表示されていた。

### 原因ファイル

`grep -rL "configSchema" extensions/*/openclaw.plugin.json` で特定：

| ファイル                                           | 問題                          |
| -------------------------------------------------- | ----------------------------- |
| `extensions/universal-skills/openclaw.plugin.json` | `configSchema` フィールドなし |
| `extensions/x-poster/openclaw.plugin.json`         | `configSchema` フィールドなし |

どちらも以前のセッションで新規作成したプラグインで、`configSchema` を付け忘れていた。

---

## 修正内容

両ファイルに空スキーマを追加（`duckduckgo` / `memory-core` / `lobster` と同じパターン）：

```json
"configSchema": {
  "type": "object",
  "additionalProperties": false,
  "properties": {}
}
```

## 検証

修正後、`openclaw --profile desktop-stack doctor` を実行してエラーが消えることを確認する。

## 今後の注意

新しいプラグイン (`openclaw.plugin.json`) を作成するときは必ず `configSchema` を含めること。設定が不要なプラグインでも空オブジェクトスキーマが必須。
