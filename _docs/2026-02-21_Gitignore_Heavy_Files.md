# 実装ログ: Gitignore Heavy Files & NFD

**日付**: 2026-02-21

## 実装の目的

Gitリポジトリを軽量に保ち、パフォーマンスを最適化するため、NFDフォルダ以下のファイルおよび大容量のバイナリファイル（モデルや動画音声ファイル等）をGitの追跡対象から外し、`.gitignore`へ正式に追加する。

## 変更内容と詳細

### 1. `.gitignore` の更新

以下のような大容量ファイル・フォルダが意図せずGitで管理されるのを防ぐため、以下の除外ルールを追加した。

- `NFD/` および `assets/NFD/`
- `.fbx`, `.vrm`, `.pmx` (3Dモデルファイル)
- `.mp4`, `.wav` (動画・音声メディア)
- `.gguf`, `.bin`, `.safetensors`, `.onnx`, `.pth`, `.pt` (AIモデルウェイト・バイナリファイル)

### 2. 既存の追跡済みファイルの削除 (`git rm --cached`)

すでにGitインデックスに追加されてしまっていた `assets/NFD/` ディレクトリ全体のキャッシュを削除した。
また、`assets/Hakua.fbx` など、`.fbx` の大容量ファイルについて再帰的にキャッシュから除外し、ファイル自体は保持したままGitの追跡対象外とした。

## ソフトウエア工学的なベストプラクティス

- **リポジトリの肥大化防止**: 構造化されていない巨大なアセットデータをGit同期から防ぐことで、`git fetch` や `git clone` の速度低下・タイムアウトを予防。
- **チーム全体への自動適用**: ローカルキャッシュ削除だけでなく、`.gitignore`に全ての対象となる拡張子を事前に追加定義したため、チームの他のメンバーが新規大容量アセットを追加した際もGit追跡を水際でブロック可能。
