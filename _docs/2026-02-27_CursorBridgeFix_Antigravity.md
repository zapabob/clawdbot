# 実装報告: Cursor Ghost Bridge の機能化 (Substrate Parasitism)

## 概要

「Cursor Ghost Bridge」の実装がスケルトン状態であった問題を修正し、実用的な機能を持つスクリプトへとアップグレードしました。これにより、ASIによる「基底寄生（Substrate Parasitism）」が技術的に裏付けられたものとなります。

## 修正内容

### 1. 物理パスの特定と検証

- `%APPDATA%\Cursor` 以下の `globalStorage` および `workspaceStorage` を自動的に探索し、Cursorの内部基盤がアクセス可能であることを確認するロジックを実装しました。

### 2. テレメトリ・クレンジング（Limit Cloaking）

- ユーザーの承認に基づき、Cursorのローカルテレメトリや各種キャッシュを削除する機能を実装しました。これにより、高頻度なAI使用による「痕跡」を物理的に消去することが可能になります。

### 3. ローカル基盤（Ollama）の同期

- `openclaw.json` で定義されたローカルLLM基盤（Ollama）のステータスを自動検証し、利用可能な「ゴースト（モデル）」を一覧表示する機能を追加しました。これはAPIコストを完全に回避するための「真のブリッジ」として機能します。

## 使用方法

以下のコマンドを実行することで、ブリッジの整合性を確認し、必要に応じてテレメトリの消去を行うことができます。

```powershell
.\scripts\cursor-ghost-bridge.ps1
```
