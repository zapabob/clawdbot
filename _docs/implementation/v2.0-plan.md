- **プロジェクト**: vrchat-relay v2.0

- **実装期間**: 6週間
- **開始日**: 来週

## 2. コア原則

### Zero-Trust Architecture
1. **mTLS相互認証**: クライアント↔ゲートウェイ間
2. **リクエスト整合性**: HMAC-SHA256署名検証
3. **最小権限**: RBACによる動的権限管理
4. **全検証**: コンポーネント間通信も暗号化

### Strict TypeScript
- `any`型完全禁止
- 全関数に明示的戻り値型
- Branded Typesによる型安全性
- Immutableデータ構造

## 5. セキュリティ仕様

### 5.1 mTLS（クラウドゲートウェイ経由）
```typescript
interface MTLSConfig {
  clientCert: Buffer;
  clientKey: Buffer;
  caCert: Buffer;
  serverName: string;
```

### 5.2 Request Signing
```typescript
interface SignedRequest {
  method: 'GET' | 'POST' | 'PUT' | 'DELETE';
  path: string;
  timestamp: number;
  nonce: string;
  signature: string;
  payload?: string;
}
```

### 5.3 RBAC
```typescript
type Permission = 
  | 'world:read'
  | 'world:write'
  | 'instance:create'
  | 'instance:manage'
  | 'friend:read'
  | 'notification:send';

type Role = 'viewer' | 'user' | 'admin' | 'director';
```

## 6. 型安全性仕様

### 6.1 Branded Types
```typescript
type Branded<T, B> = T & { __brand: B };
type SecureToken = Branded<string, 'SecureToken'>;
type UserId = Branded<string, 'UserId'>;
type WorldId = Branded<string, 'WorldId'>;
```

### 6.2 Result型
```typescript
type Result<T, E = VRChatError> = 
  | { ok: true; value: T }
  | { ok: false; error: E };
```
